# 个股详细分析报告 — 功能规格书

## 概述
用户输入股票代码/名称后，生成一份专业量化分析报告。不是简单的评分数字，而是**多模块深度拆解**，让用户理解"为什么这个分数"和"现在该怎么做"。

当前 `stock_app.py` 已经有完整实现，入口函数是 `run_full_analysis(code)`，输出到 `temp_report.txt` 并通过GUI展示。

---

## 输入方式
- 股票代码输入框 + 「分析」按钮
- 「今日信号」弹窗中每只股票旁边的「分析」快捷按钮
- 自选股一键分析按钮

## 报告包含模块（按顺序）

### 模块1: 基本面概览
- 公司全称、行业、主营业务（截取前120字）
- 总市值、流通市值
- 市盈率(PE)
- 交易所、注册资本
- 所属板块标签（如"半导体""百元股""大盘成长"）
- 数据来源: `get_stock_sector_info(code)` → emweb.securities.eastmoney.com

### 模块2: 今日行情
- 现价、涨跌幅%（颜色标识红涨绿跌）
- 开盘/最高/最低
- 量比、换手率、成交额、成交量
- 市盈率、涨停价、跌停价
- 数据来源: `_fetch_tencent_kline(code)` 最新一行 + 腾讯qt.gtimg.cn实时行情

### 模块3: 综合评分（核心）
5维拆解，每个维度显示具体得分/满分 + 评分理由：
- **D1 趋势质量 (0-30)**: 均线排列 + 回调深度 + 止跌形态
- **D2 量价健康 (0-30)**: 缩量程度 + 价量背离 + 换手率
- **D3 资金信号 (0-20)**: 主力净流入方向 + 趋势 + 可靠性
- **D4 板块共振 (0-20)**: 同花顺行业排名/ETF共振 + 相对强度
- **D5 风险惩罚 (-20~0)**: 高位风险/见顶信号/一字板/ST
- 总分 + 等级（跟庄买点/关注蓄势/观望/偏弱/回避）+ 建议操作
- 数据来源: `calc_score()` — 注意传递 `fast_mode=False` 获取完整D4

### 模块4: 信号详情
逐条列出评分过程中产生的所有信号（带 +/- 标识）：
- ✅ 正向信号（如"从高点回调X%，进入低吸区间"）
- ❌ 负向信号（如"上影线过长，高位压力"）
- 每条都有具体数值，不是模糊描述
- 数据来源: `calc_score()` 返回的 reasons 列表

### 模块5: 近5日主力资金
表格展示：
- 日期 | 主力净流入(亿) | 超大单净额 | 大单净额
- 来源标注（东方财富/估算）
- 数据来源: `_fetch_em_fund_flow_raw(code)` → emdatah5.eastmoney.com

### 模块6: 技术形态检测
- 今日形态: 锤子线/十字星/放量冲高回落/缩量小阴线等
- K线特征: 上影占比/下影占比/实体大小/振幅
- 数据来源: `label_patterns(df)` 返回的形态标签

### 模块7: 历史回测
当前触发的技术形态在历史上出现后的次日表现：
- 形态名称 | 样本数 | 胜率 | 期望收益 | 盈亏比
- 最近3次相似形态日期及次日涨跌幅
- 数据来源: K线历史数据 + 形态匹配算法

### 模块8: 持仓对比（如有持仓）
- 显示当前该股持仓: 股数、成本、现价、浮盈/浮亏
- 数据来源: `stock_data/paper_trading/account.json`

---

## GUI展示要点
- 用 `tkinter.scrolledtext` 展示富文本报告
- 各模块用分割线隔开，模块标题加粗
- 涨跌用颜色: 涨红跌绿(A股惯例)、涨绿跌红(国际惯例) — 用户用红涨绿跌
- 评分等级用emoji+文字: 🟢跟庄买点 🟡关注蓄势 ⚪观望 🟠偏弱 🔴回避

## 核心数据调用链
```
用户输入code
  → get_market_regime()              # 市场状态
  → get_stock_sector_info(code)      # 基本面
  → _fetch_tencent_kline(code)        # K线
  → _fetch_em_fund_flow_raw(code)     # 资金流向
  → label_patterns(df)                # 形态标注
  → calc_score(latest, df, ff, ..., fast_mode=False)  # 评分
  → 历史回测（形态匹配）
  → 格式化输出 → 展示
```

## 重要提示
1. `fast_mode=False` 是必须的 — 这样D4才会用同花顺真实行业排名，不做默认10分
2. 所有HTTP请求都要有try/except保护，失败时显示"数据暂不可用"而不是崩溃
3. 报告生成在后台线程执行（`threading.Thread`），避免阻塞GUI
4. 如果用户输入了非CSI300股票代码，仍然要尝试分析（用腾讯API数据回退）
5. 已有完整实现在 `stock_app.py:run_full_analysis()` 和 `_display_report()`，可参考但UI可以重新设计
