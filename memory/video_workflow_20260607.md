# 视频制作工作流 - 2026-06-07

## 默认范围

- 后续视频默认不生成字幕。
- 后续视频默认不生成 BGM、不下载 BGM、不切 BGM。
- 除非用户明确说要字幕或 BGM，否则只做：文案、画面方案、素材建议、草稿/动画、剪映可编辑结构。

## 核心原则

- 用户手改过的剪映草稿，先复制再处理，绝不直接动原件。
- 剪映 6.0+ 保存后的 `draft_content.json` 可能是加密/编码串，不能强行自动改。能不能改，先看文件是否为明文 JSON。
- 画面先定动画和结构，再进剪映手改文字。手改保存后，代码批量回改风险很高。
- 不追求花哨模板动画，优先做镜头感、中心线、层次和稳定构图。

## 草稿处理

1. 先确认源草稿路径和名称。
2. 复制一个新草稿，命名带 `_codex_copy` 或任务后缀。
3. 检查 `draft_content.json`：
   - 明文 JSON：可以用脚本改图层、关键帧、动画。
   - 加密/编码串：不强改，只能给手动参数或基于脚本重新生成。
4. 如果用户已手动改好文字/位置，默认只做可安全处理的内容。

## 图片动画规则

- 背景图：铺满全屏，慢慢平移，不缩放。
- 主图/展示图：与背景图方向相反移动。
- 主图中心点沿画面水平中线移动，720p 下 `y=360`。
- 主图用当前大小作为起点，轻微放大即可，推荐 `scale 0.9 -> 0.96`。
- 不做从很小快速放大的副图动画。
- 不做快速闪图，不做过度透明度花活。
- 多主图画面：只让当前叙事主图动，其他图轻微或静止，别抢镜。

## 音乐处理规则

- 默认不处理 BGM。
- 如果用户明确要求 BGM，再处理。
- 不用“民族风素材库”方向，容易土。
- 不搜 `ethnic`、`oriental`、`chinese traditional` 作为主关键词。
- 用户偏好：要史诗感，但不能太像游戏/电影预告片盖过旁白。
- 参考方向：John Dreamer - Becoming a Legend 这种史诗推进，但音量要压。

## 质量判断

- 画面：别乱、别飘、别副图抢主图。
- 动画：少即是多，慢推、反向移动、轻微放大最稳。
- 音乐：别民族素材味，别廉价国风，别旅游宣传片感。
- 旁白永远是主角，BGM 只托情绪。
