## 四、改造建议（Claude 的推荐方案）

### 核心思路：V9评分 + 重写执行层 = 新策略

**一句话**：废弃 V2.3，用 V9 的 7 维评分引擎，加上全新的交易执行层。

### 为什么

| 对比维度 | V2.3 | V9 | 谁赢 |
|---------|------|-----|:--:|
| 评分维度 | 5维 | 7维 | V9 |
| 自适应权重 | 无 | 牛市/震荡/熊市 | V9 |
| 资金强度归一化 | 无 | 5日净流入/日均成交额 | V9 |
| 板块动量 | 弱 | 行业内排名+动量+热度 | V9 |
| 个股相对强度 | 无 | 5日涨幅 vs CSI300 | V9 |
| 区分度 | 低 | 高（D5从0%复活，D2从7%→8+种值） | V9 |
| 实盘执行层 | ✅ 完整 | ❌ 无 | V2.3 |
| 出场规则 | ✅ 4重 | ❌ 无 | V2.3 |
| 回测可靠性 | ❌ 过拟合 | 待验证 | — |

### 具体做法

```
新策略 = V9评分引擎 + 新执行层

【保留】V9 评分引擎（不改）：
  engine/constants.py          ← 所有参数
  engine/score_calculator.py   ← calc_score_v9()
  engine/dimensions/d1~d7.py   ← 7个维度
  engine/market_regime.py      ← 市场判定

【重写】执行层：
  1. 入场规则：评分>阈值（牛市55/震荡60/熊市70）+ 板块共振确认（D2>10）
     不用 top_n 限制，凡是达标的都进候选池
  
  2. 仓位管理：用 V9 的 ATR+Kelly 混合算法（已有 position_sizer.py）
     单票上限30%，现金>=5%，ATR>7%直接跳过
  
  3. 出场规则：
     - -5% 硬止损（保留V2.3的）
     - 动态移动止盈：浮盈>15%收紧到1.5xATR / >8%保本 / >3%成本线
     - 10天到期 + 动量衰减双重保护
  
  4. 换仓机制：候选票比最弱持仓高5分以上自动换

  5. 回测验证：Walk-forward，覆盖2024-01→2026-05，至少涵盖一轮牛熊
```

### 预期效果

- V9 扫描能稳定选出瑞芯微(+10%)、长电科技(+50%)这种强势股
- 加上执行层后，不再只是"推荐"而是"买多少、什么价买、什么价卖"
- 回测覆盖更长时间段，避免V2.3的过拟合问题

## 五、给你的建议（Codex）

上面这些都是我（Claude）的分析和建议。你可以：

1. **先看执行层缺失**：V9 的 `engine/` 目录下评分引擎是完整的，缺的是策略文件。需要新建一个类似 `strategy_runner.py` 的文件，把评分、入场、仓位、出场、换仓串起来。

2. **V2.3 的出场规则值得参考**：`aggressive_strategy.py` 里的 4 重出场（硬止损/移动止盈/动量衰减/到期）逻辑是对的，只是评分选股不行。

3. **别动评分维度**：V9 的 7 维刚刚调优过（5 月 22 日），34 项验证全通过。D1/D2/D5 的优化是有效的——D5 从死维度复活，D2 区分度从 7% 提升到 8+ 种值。

4. **最需要你动手的是**：新建一个策略主文件，组装 V9 引擎 → 每天扫描 → 筛选达标股 → 算仓位 → 生成交易指令 → 记录日志。现有的 `position_sizer.py` 和 `account_manager.py` 可以直接用。
